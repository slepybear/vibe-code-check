## 🔴 发现的 Bug 清单

### Bug #1：vercel.json 路由配置错误（P0 - 严重）

**问题描述：**
[vercel.json](vercel.json) 配置了 SPA 重写规则，导致所有子路由都被重定向到首页。

**影响范围：**
- ❌ 候选人访问 `/candidate?problemId=xxx` → 显示首页
- ❌ 面试官访问 `/interviewer` → 显示首页
- ❌ **核心功能完全不可用**

**根因分析：**
```json
// 错误配置 ❌
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```
这是 Create React App 项目需要的配置，但本项目使用 Next.js 14 静态导出模式。

**修复方案：**
✅ **删除 vercel.json 文件**

Next.js 静态导出生成的 `out/` 目录已包含完整的路由结构：
```
out/
├── index.html           (首页)
├── candidate/index.html (候选人页)
└── interviewer/index.html (面试官页)
```

Vercel 会自动识别 Next.js 输出格式，无需额外配置。

**修复状态：** ✅ 已完成

### Bug #2：useSearchParams() SSR/CSR 不一致（P1 - 高）

**问题描述：**
在静态导出模式下，`useSearchParams()` 在首次渲染时可能导致 `problemId` 为空。

**影响场景：**
- 用户第一次点击链接进入时闪烁或显示错误页面

**修复方案：**
✅ 将 problem 查找逻辑移入 `useEffect`

**修改文件：** [app/candidate/page.tsx#L19-L28](app/candidate/page.tsx#L19-L28)

```tsx
// 修改前 ❌
const [problem, setProblem] = useState(problems.find(p => p.id === problemId) || null);

// 修改后 ✅
const [problem, setProblem] = useState<Problem | null>(null);

useEffect(() => {
  if (problemId) {
    const foundProblem = problems.find(p => p.id === problemId);
    setProblem(foundProblem || null);
  } else {
    setProblem(null);
  }
}, [problemId]);
```

**修复状态：** ✅ 已完成

### Bug #5：缺少环境检测提示（P2 - 中）

**问题描述：**
面试官在本地开发环境生成的链接（localhost:3000）无法分享给候选人使用。

**修复方案：**
✅ 添加环境检测和控制台警告 + UI 提示

**修改文件：** [app/interviewer/page.tsx](app/interviewer/page.tsx)

```tsx
// 控制台警告
if (baseUrl.includes('localhost') || baseUrl.includes('127.0.0.1')) {
  console.warn('⚠️ 当前是本地开发环境，此链接仅在本机可用！');
}

// UI 提示（仅 localhost 显示）
{window.location.origin.includes('localhost') && (
  <div className="bg-yellow-900/20 border border-yellow-600/30 rounded-lg p-3">
    <p className="text-yellow-400 text-xs">
      ⚠️ 当前为本地开发环境，此链接仅本机可用。
      <br />部署到生产环境后再分享给候选人。
    </p>
  </div>
)}
```

**修复状态：** ✅ 已完成
