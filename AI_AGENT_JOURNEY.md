# 我用 AI Agent 2 天做了个面试工具

## 为什么做这个？

面试的时候我发现一个很无语的事：现在都 2026 年了，很多公司还在用 LeetCode 刷题来面试程序员。

但问题是，LeetCode 根本测不出一个开发者的真实水平。特别是现在大家都用 AI 写代码了，面试更应该看的是：**你能不能用 AI 把一个真实需求落地**。

所以我想做一个面试工具——让候选人现场做一个真实的业务场景，而不是背算法题。但一个人从零开始搞这种东西，光搭框架就得折腾好几天。

## AI Agent 帮我干了什么

我用了 Trae（字节跳动的 AI 编程工具），2 天就把 VibeCode Check 搞出来了：

**第一天：**
- 搭 Next.js 项目，配 Tailwind
- 接 Monaco Editor（代码编辑器）
- 做 iframe 实时预览
- 写 9 道面试题（初级/中级/高级各 3 道）

**第二天：**
- 做面试官面板 + 候选人编码界面
- 加图片上传功能
- 代码导出（JSZip 打包）
- 部署到 Vercel

以前这种活我自己干至少两周，这次两天搞定。而且代码质量比我一个人写还好——Agent 会提醒我注意边界情况、帮我优化结构。

## 还有个坑没填完

工具是能用了，但有个 bug 一直没彻底解决：**图片有时候显示不出来**。

这个问题分两部分：

### 1. 代码逻辑问题

我的预览是用 iframe 沙箱跑的，图片上传后转成 base64 存在内存里。然后在生成预览 HTML 的时候，把代码里的图片路径替换成 base64 内容。

现在的替换逻辑只处理了一种情况：

```html
<img src="logo.png">  ✅ 这个能替换
```

但这几种就不行了：

```css
background-image: url('logo.png');  ❌ CSS 背景图没处理
```

```javascript
img.src = 'logo.png';  ❌ JS 动态赋值没处理
```

```html
<img src=./logo.png>  ❌ 不带引号的也没处理
```

### 2. 环境差异问题

更麻烦的是，同样的代码在不同环境下表现不一样：

- 在豆包的 Code Interpreter 里跑，图片能正常显示
- 部署到 Vercel 上之后，有些图片就显示不了了
- 本地开发环境又和线上环境不一样

我怀疑是 iframe 沙箱的安全策略、base64 编码处理，或者不同浏览器的兼容性问题。这个问题说大不大说小不小，但修起来挺烦的——要考虑的情况太多了。

## 我觉得理想的 Agent 应该怎么做

与其让 Agent 一个个去匹配各种写法，不如换个思路：

**方案一：拦截层方案**

不在字符串层面做替换，而是在 iframe 里拦截所有资源请求。不管代码怎么引用图片，只要请求过来了，我就从虚拟文件系统里把对应的 base64 返回去。这样不管你是 `<img src>`、`url()` 还是 `img.src =`，统统都能处理。

**方案二：多环境测试**

Agent 能自动在不同环境里测试：
- 本地开发环境
- 线上部署环境（Vercel）
- 不同浏览器（Chrome、Firefox、Safari）
- 不同 AI 平台的 Code Interpreter 环境

这样就能自动发现环境差异问题，而不是等用户反馈才知道。

**方案三：静态分析 + 可视化**

Agent 扫一遍代码，把所有涉及图片的地方标出来——哪些能正常显示、哪些会挂掉，用不同颜色高亮。这样开发者一眼就能看出问题在哪，而不是等到预览里才发现图片没了。
